## 2. 产品定位

**一句话定位**：一个免费、开源、以投资系统化为核心的 Web 深度学习平台，通过强制评测与 DeepSeek AI 点评形成学习闭环。

**核心价值**：
1. **体系化投资教育**：从财务基础到组合管理，构建完整投资框架。
2. **强制诊断与路径**：不通过测评无法进入下一阶段，确保基础扎实。
3. **DeepSeek AI 即时反馈**：开放题、项目作业秒级点评，降低师资成本。
4. **全开源生态**：课程内容、课程数据、工具链全部开放，社区共建。
5. **零门槛**：无付费、无发帖限制，最大化降低参与成本。

## 3. 目标用户

| 用户类型 | 特征 | 学习目标 | 使用方式 |
|---|---|---|---|
| **投资新手** | 刚接触股票/基金，看过零散科普 | 建立正确投资观，避免追涨杀跌 | 从 L1 开始，按路径逐关解锁 |

| **长期价值投资者** | 认同价值投资，想深化研究能力 | 完善研究清单和决策流程 | 重点学习 L4-L5 项目与案例 |

## 4. 投资领域课程大纲

### 4.1 课程结构总览

| 阶段 | 阶段名称 | 课时数 | 预计学习时间 | 解锁条件 |
|---|---|---|---|---|
| L0 | 平台导学与学习法 | 3 课时 | 30 分钟 | 注册即解锁 |
| L1 | 投资思维与市场认知 | 12 课时 | 4-5 小时 | 通过 L1 入学测评 |
| L2 | 财务会计与报表分析 | 18 课时 | 8-10 小时 | 通过 L2 入学测评 |
| L3 | 估值方法与安全边际 | 15 课时 | 7-9 小时 | 通过 L3 入学测评 |
| L4 | 行业与公司研究 | 14 课时 | 8-10 小时 | 通过 L4 入学测评 |
| L5 | 投资组合与风险管理 | 12 课时 | 6-8 小时 | 通过 L5 入学测评 |
| L6 | 行为金融与投资系统 | 10 课时 | 5-6 小时 | 通过 L6 入学测评 |
| 实战 | 投资研究项目 | 6 个项目 | 20-40 小时 | 按阶段解锁 |

### 4.2 课程单元标准格式

每个课时必须包含以下 6 个部分：

1. **学习目标**：学完能做什么
2. **核心概念**：图文/视频，5-15 分钟
3. **案例解析**：至少 1 个真实案例
4. **思考练习**：1-3 道开放或半开放题
5. **延伸阅读/资料**：论文、书籍章节、公开报告
6. **课后讨论题**：链接到社区对应话题

### 4.3 L0：平台导学与学习法

1. **课程介绍**：IdleLearn 怎么用、为什么从投资开始
2. **有效学习法**：主动回忆、间隔重复、费曼技巧
3. **如何使用 DeepSeek AI 点评**：提交标准、反馈解读、如何迭代

### 4.4 L1：投资思维与市场认知

| 课时 | 标题 | 核心内容 | 课后练习 |

## 5. 强制评测系统设计

### 5.1 核心规则

- **不可跳过**：每个阶段入口必须完成评测才能解锁。
- **自适应出题**：根据答题表现动态调整难度。
- **通过标准**：百分制达到 70 分视为通过。
- **重考机制**：未通过需等待 24 小时才能重考，期间系统推荐复习内容。
- **阶段内复测**：每个阶段结束时设置结业测，不通过仍可进入下一阶段，但标记为“待巩固”。

### 5.2 题型设计

| 题型 | 数量/阶段 | 目的 |
|---|---|---|
| 概念选择题 | 8-10 题 | 检验基础概念理解 |
| 判断题 | 3-5 题 | 识别常见误区 |
| 案例分析题 | 2-3 题 | 检验知识应用能力 |
| 计算/推导题 | 1-2 题 | 检验量化分析能力 |
| 开放论述题 | 1 题 | 由 DeepSeek AI 点评评分 |

### 5.3 测评结果应用

- 推荐起始阶段（新手可从 L1，有经验者可从 L2/L3）。
- 生成能力雷达图。
- 自动推荐薄弱知识点课程。
- 影响学习路径中的重点内容排序。

## 6. DeepSeek AI 点评系统设计

### 6.1 点评范围

- 阶段项目作业
- 开放论述题
- 研究报告
- 投资备忘录
- 社区中的优质长文（可选）

### 6.2 点评流程

1. 用户提交作业（支持 Markdown、PDF、图片）。
2. 系统进行格式校验与查重。
3. DeepSeek 基于评分 rubric 生成结构化反馈。
4. 用户查看反馈并修改。
5. 用户可重新提交，DeepSeek 对比前后版本给出改进评价。
6. 优秀作业可进入社区精选（需用户授权）。

### 6.3 点评维度

| 维度 | 权重 | 说明 |
|---|---|---|
| 完整性 | 20% | 是否覆盖题目要求的全部要点 |
| 逻辑性 | 25% | 论证是否清晰、结构是否合理 |
| 准确性 | 25% | 概念、数据、推理是否正确 |
| 深度 | 15% | 是否有独立见解，而非照搬课程 |
| 表达 | 10% | 语言是否清晰、专业 |
| 可执行性 | 5% | 投资相关建议是否具体、可落地 |

### 6.4 反馈格式

```markdown
## 总体评分：82/100

### 优点
1. ...
2. ...

### 待改进
1. ...
2. ...

### 关键问题
- 问题 1：...（附改进建议）
- 问题 2：...

### 下一步建议
建议先复习 L2-07《盈利能力比率》，然后补充 ROIC 与 ROE 的对比分析。
```

### 6.5 人工复核机制

- 用户对 DeepSeek 点评有异议时可申请人工复核。
- 系统随机抽取 5% 的 AI 点评进行人工抽检。
- 建立“点评质量反馈”按钮，用于持续优化 prompt 与模型。

### 6.6 技术实现

- 接入 DeepSeek API（如 DeepSeek-V3/DeepSeek-R1）。
- 使用结构化输出（JSON Schema）确保反馈格式一致。
- Prompt 中注入课程知识点和评分标准。
- 对长文档先分块摘要，再整体点评。
- 成本控制：限制每次作业输入 token，对超长文档要求用户精简。

## 7. 社区设计（无发帖门槛）

### 7.1 社区原则

- **无发帖门槛**：注册用户即可发帖、评论、点赞。
- **按课程/项目组织讨论**：每个课时和项目都有独立讨论区。
- **质量优先**：通过算法排序让高质量内容获得更多曝光。
- **友好氛围**：明确社区规范，禁止荐股、人身攻击、广告。

### 7.2 社区模块

| 模块 | 功能 |
|---|---|
| 课时讨论区 | 每节课下方的提问与讨论 |
| 项目展示区 | 用户自愿公开的项目作业 |
| 问答广场 | 跨课程综合问题 |
| 学习小组 | 按兴趣自发组建（如“银行股研究小组”） |
| 精华区 | 优质笔记、案例分析、学习心得 |
| 动态流 | 关注用户的学习动态 |

### 7.3 内容排序机制

- 默认排序：综合质量分（点赞、收藏、回复质量、作者学习等级）。
- 时间排序：最新回复。
- 热度排序：近 7 天互动量。

### 7.4 内容治理

- AI 预审：发帖前自动检测违规内容。
- 用户举报。
- 管理员/版主复核。
- 重复低质内容降权。
- 严重违规禁言/封号。

### 7.5 激励机制

- 优质内容获得“学习贡献者”徽章。
- 被收录精华区的内容可显示在课程“学员佳作”中。
- 高贡献用户可申请成为课程维护者。

## 8. 开源内容策略

### 8.1 开源范围与协议

| 内容 | 协议 | 存储位置 |
|---|---|---|
| 课程大纲与课时内容 | CC-BY-SA 4.0 | `github.com/ubiy123/idlelearn-courses` |
| 练习题与项目模板 | CC-BY-SA 4.0 | `idlelearn-courses/exercises/` |
| 评测题库 | CC-BY-SA 4.0 | `idlelearn-courses/assessments/` |
| AI 点评 Prompt 模板 | MIT | `github.com/ubiy123/idlelearn-ai` |
| 前端/后端代码 | MIT | `github.com/ubiy123/idlelearn` |
| 课程数据（JSON/YAML） | CC-BY-SA 4.0 | `idlelearn-courses/data/` |

### 8.2 开源协作流程

1. 课程内容以 Markdown/YAML 格式存储。
2. 每个课程一个目录，包含：
   - `README.md`：课程说明
   - `lessons/`：课时内容
   - `exercises/`：练习题
   - `projects/`：项目说明与评分标准
   - `assets/`：图片、数据文件
3. 通过 Pull Request 提交内容改进。
4. 由领域专家审核后合并。
5. 网站自动同步 GitHub 主分支内容。

### 8.3 内容贡献者激励

- GitHub 贡献者名单展示在网站“贡献者墙”。
- 重要贡献者获得网站管理员权限。
- 每年评选“年度课程贡献者”。

## 9. 技术架构

### 9.1 前端

- **框架**：Next.js 14 App Router
- **样式**：Tailwind CSS + shadcn/ui
- **状态管理**：Zustand
- **富文本/笔记**：TipTap
- **公式渲染**：KaTeX（财务/估值公式）
- **图表**：ECharts

### 9.2 后端

- **框架**：Python FastAPI
- **数据库**：PostgreSQL
- **缓存**：Redis
- **任务队列**：Celery + Redis
- **对象存储**：MinIO
- **搜索**：Meilisearch
- **内容同步**：GitHub Webhook → 自动拉取课程仓库

### 9.3 DeepSeek AI 点评服务

- 独立微服务，与主服务解耦
- 接入 DeepSeek API
- 使用结构化输出确保反馈格式统一
- 异步处理：用户提交后进入队列，完成后通知

### 9.4 部署

- Docker Compose 本地开发
- Kubernetes 生产环境
- CI/CD：GitHub Actions
- 监控：Prometheus + Grafana + Sentry

## 13. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 强制评测导致用户流失 | 注册转化率下降 | 测评前提供样题，测评失败给明确复习路径 |
| DeepSeek 点评质量不稳定 | 用户不信任 | 公开点评标准，允许申诉，人工抽检 |
| 开源内容质量参差不齐 | 课程权威性受损 | PR 审核机制，专家把关 |
| 无发帖门槛导致低质/违规内容 | 社区体验差 | AI 预审 + 举报 + 排序降权 |
| 投资内容合规风险 | 法律问题 | 全程免责声明，禁止荐股，内容合规审核 |
| 全免费模式不可持续 | 项目停滞 | 寻求企业赞助、教育 grants、志愿者贡献 |
